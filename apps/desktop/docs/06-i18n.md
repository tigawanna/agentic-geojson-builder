# Internationalization (i18n)

Powered by [i18next](https://www.i18next.com/) + `react-i18next`.

## Adding a key

1. Edit [`src/renderer/src/i18n/locales/en.json`](../src/renderer/src/i18n/locales/en.json):

   ```json
   {
     "home": {
       "welcome": "Hello, {{name}}"
     }
   }
   ```

2. Add the same key to every other locale (at minimum: `hi.json`). CI should
   fail if a key is missing — you can add a lint check with
   [`i18next-parser`](https://www.npmjs.com/package/i18next-parser) later.

3. Use it:

   ```tsx
   const { t } = useTranslation()
   return <p>{t('home.welcome', { name: user.name })}</p>
   ```

## Adding a new locale

1. Create `src/renderer/src/i18n/locales/<code>.json` — copy `en.json` and
   translate.
2. Register it in [`src/renderer/src/i18n/index.ts`](../src/renderer/src/i18n/index.ts):

   ```typescript
   import fr from './locales/fr.json'

   void i18n.init({
     resources: {
       en: { translation: en },
       hi: { translation: hi },
       fr: { translation: fr },
     },
     // ...
   })

   export const supportedLanguages = [
     { code: 'en', label: 'English' },
     { code: 'hi', label: 'हिन्दी' },
     { code: 'fr', label: 'Français' },
   ] as const
   ```

The `<SettingsPage>` language selector reads from `supportedLanguages` and
auto-updates.

## Persisting language choice

The language picker currently changes `i18n.language` in memory. To persist
across restarts, save it via IPC:

```tsx
onChange={async (e) => {
  await i18n.changeLanguage(e.target.value)
  await ipcInvoke('store:set', { key: 'language', value: e.target.value })
}}
```

And load it at boot inside `i18n/index.ts` (wrap `init` in an async function
that first awaits `ipcInvoke('store:get', { key: 'language' })`).
